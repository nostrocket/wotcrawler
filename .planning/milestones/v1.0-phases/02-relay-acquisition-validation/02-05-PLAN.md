<objective>
Close the three fetch.rs completeness/safety BLOCKERs (CR-02, CR-03, CR-04) that
undermine the phase's "completely" guarantee. The pagination loop currently
(a) loses events at the cap boundary second (exclusive `oldest - 1`),
(b) has no page budget or progress guard so an adversarial relay ignoring `until`
loops forever with unbounded memory, and (c) never constructs `RelayError::FetchTimeout`
because the SDK returns a partial `Ok` on timeout — so timed-out windows are
silently recorded as complete (silent data loss).

Purpose: restore RELAY-03 completeness and adversarial-relay safety so the
acquisition half emits complete follow-list windows and requeues (rather than
silently drops) timed-out or hostile windows.

Output: corrected `page_back`/`paginate_chunk`/`fetch_complete_with_timeout` in
`src/relay/fetch.rs`, the new `MAX_PAGES_PER_CHUNK` constant, plus new tests that
FAIL before the fix and PASS after.
</objective>
