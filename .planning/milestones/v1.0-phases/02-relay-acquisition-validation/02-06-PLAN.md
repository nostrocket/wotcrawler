<objective>
Close CR-01 (BLOCKER) — dedup-before-verify in the ingest orchestrator. Today
`seen.insert(event.id)` runs BEFORE `verify::accept`, so a hostile relay can
id-squat: it sends a forged event carrying a victim's genuine event id first,
the forged id enters the seen-set, verification fails (forged), and the genuine
copy from an honest relay is then skipped as a "duplicate". This inverts
INGEST-02's purpose — duplicate suppression becomes a censorship primitive.

Purpose: make dedup follow verification so only VERIFIED ids occupy the seen-set,
restoring INGEST-02 as a correctness guarantee rather than an attack surface.

Output: reordered `ingest_events` loop in `src/ingest/mod.rs` and a new id-squat
attack test that fails before the fix and passes after.
</objective>
