<objective>
Close CR-06 (BLOCKER) and WR-02 (WARNING) in the NIP-11 path. Today
`reqwest::Client::new()` carries no timeout (violates project Pitfall 9 — every
network fetch carries a deadline) and `resp.text()` reads an unbounded body, so a
hostile relay can hang the crawler or stream an arbitrarily large body to exhaust
memory. Separately, `clamp_limit` accepts any positive i32, so a relay advertising
`max_limit=2000000000` restores Pitfall 1 (a cap so large the pagination loop
treats one EOSE window as complete).

Purpose: make NIP-11 acquisition deadline-bounded, memory-bounded, and immune to
an absurd advertised cap so RELAY-02 limits are trustworthy inputs to the
pagination planner.

Output: a shared, timeout-configured reqwest client built once via LazyLock, a
body-size bound `MAX_NIP11_BYTES`, an upper clamp `MAX_ADVERTISED_LIMIT`, and
tests for the clamp and (where unit-testable) the body bound.
</objective>
