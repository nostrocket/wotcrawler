<objective>
Define the Phase 1 schema as a single idempotent versioned migration: `pubkeys` (surrogate bigint id, 32-byte bytea pubkey, status, freshness + churn columns, applied-event columns), `follows` (bare directed edges on bigint ids with a self-follow CHECK), the supporting indexes, the three stable contract views, and `COMMENT ON` statements making the contract introspectable. Cover it with integration tests proving migrate-from-empty, re-run idempotency, schema shape, and contract-view presence/comments.

Purpose: The schema IS the project boundary and the spam layer's public API (GRAPH-01, GRAPH-04). It must migrate idempotently (success criterion 1) and expose a stable, documented contract surface (success criterion 4). Getting columns right now avoids migrations on hundreds of millions of rows later — so FRESH-01/FRESH-03/INGEST-03 columns ship now (D-07, D-09, D-10) even though consumed in later phases.
Output: `migrations/0001_graph_schema.sql` plus migration/contract integration tests.
</objective>
