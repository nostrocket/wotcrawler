## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| relay events → crawl driver | untrusted relay events enter; all validation is delegated to the existing ingest gate (verify/dedup/newest-wins) — the driver adds none |
| crawl driver → graph state | the apply seam mutates durable edges + freshness; correctness gates GRAPH-02 |
| concurrent batch workers → DB | bounded by a semaphore; the DB queue is the only growable structure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | edge-diff transactional integrity (GRAPH-02) | mitigate | apply path delegates to the proven atomic `apply_follow_list`; `apply_diff_adds_and_removes` + `same_event_zero_touch` verify add/remove deltas + zero-touch idempotency against real events |
| T-03-10 | Spoofing | forged/unsolicited relay events | mitigate | driver runs the raw union through `ingest_events` (verify + solicited-author gate) ONCE; no validation bypass path added (Pitfall: do not skip the gate) |
| T-03-11 | Denial of service | unbounded in-flight memory/relay load (CRAWL-04) | mitigate | `Semaphore` permit cap; queue lives in the DB not memory; `bounded_concurrency` test asserts the cap holds |
| T-03-12 | Elevation of privilege | crawling spam islands (CRAWL-02) | mitigate | reachability is structural (upsert-on-followee is the only insertion path besides the anchor); `spam_island_never_crawled` proves an unreferenced pubkey is never fetched; no `reachable`/recursive-CTE predicate |
| T-03-13 | Repudiation | terminal status missing timestamp (FRESH-01) | mitigate | every terminal path routes through a stamping write; `last_fetched_at_stamped_on_terminal` asserts non-NULL across fetched/not_found/failed |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages (RESEARCH audit: not applicable) |
