<objective>
Build the DB-resident BFS frontier primitives: seed the anchor (D-03), claim
batches with `FOR UPDATE SKIP LOCKED` in a short lease transaction (D-04/D-07),
reclaim crash-orphaned `in_progress` rows at startup (D-06), and resolve
transient-vs-terminal retry status with `fetch_attempts` (D-09/D-10/D-11). These
are the queue/lease mechanics that make CRAWL-01/02/03 hold and FRESH-01 stamp on
every terminal path.

Purpose: The frontier IS `pubkeys.status` (D-01) — these functions are the only
custom queue logic; the hard primitives (`upsert_pubkey`, `set_fetch_status`,
`apply_follow_list`) already exist and are consumed, not rebuilt. CRAWL-02
reachability is structural: a pubkey is only ever a row because `upsert_pubkey`
landed it as a followee of someone already crawled (D-02) — no runtime reachability
predicate is added.

Output: `src/crawl/frontier.rs`, `src/crawl/mod.rs` scaffold, `src/lib.rs`
registration, an optional `fetch_attempts` helper in `src/store/pubkeys.rs`, and
the filled `tests/frontier.rs` claim/lease/reachability/crash-resume/terminal-stamp
tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| crawl driver → pubkeys queue | the lease/claim flips status and stamps timestamps; correctness here gates CRAWL-03 crash safety |
| concurrent workers → same row | two workers must never claim the same row |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | concurrent claim race | mitigate | `FOR UPDATE SKIP LOCKED` claim in a short txn; `skip_locked_no_double_claim` two-pool test proves no double-claim |
| T-03-05 | Denial of service | long-held claim transaction | mitigate | Claim is its own short transaction; lock released at commit, fetch runs lock-free (D-04, Pitfall 6) |
| T-03-06 | Repudiation | terminal status without timestamp | mitigate | `requeue_or_fail` routes `failed` through a stamping write (FRESH-01, Pitfall 5); `requeue_at_cap_sets_failed_and_stamps` asserts `last_fetched_at` non-NULL |
| T-03-07 | Denial of service | unbounded retry loop on a flaky relay | mitigate | `fetch_attempts` cap → terminal `failed` (D-09, Pitfall 7) |
| T-03-08 | Elevation of privilege | re-fetching completed work (CRAWL-03) | mitigate | claim selects only `discovered`; `claim_never_returns_fetched` test asserts a `fetched` row is never claimed (Pitfall 3) |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages (RESEARCH audit: not applicable) |
