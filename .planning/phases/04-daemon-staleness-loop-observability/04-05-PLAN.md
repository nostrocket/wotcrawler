<objective>
Wire the whole daemon together (OPS-01/OPS-02 + OBS surface activation): `src/main.rs` parses `--config` (clap), loads + validates config (exiting non-zero on failure before any crawl work), and calls `daemon::run`. `daemon::run` inits tracing, installs the Prometheus recorder, connects + migrates the DB, builds the production `fetch_union` closure (fan out `acquire_validated_lists_client` per curated relay, concat raw events), creates the shared `CancellationToken`, spawns the signal listener (SIGTERM+SIGINT → cancel), starts the axum `/metrics`+`/health/*` server with graceful shutdown, and spawns the continuous loop + staleness + in-run-reclaim + sampler + progress-summary tasks — all selecting on the token — then awaits clean shutdown.

Purpose: This is the keystone that turns the library into the single unattended `crawler` daemon binary (OPS-01) and proves the graceful-shutdown contract (OPS-02).
Output: A runnable `crawler` binary; the OBS endpoints serve live data; signals drain cleanly.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → crawler process | `--config` + env supply anchor/relays/DB URL; validated fail-fast |
| OS signals → daemon | SIGTERM/SIGINT must drain, not abort mid-write |
| curated relays → fetch_union | untrusted relay events enter; ingest gate (Phase 2) verifies before apply |
| Prometheus/operator → observability HTTP | network clients hit /metrics + /health |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Tampering/Repudiation | SIGTERM aborts workers → orphaned in_progress leases | mitigate | signal → token → loop drains in-flight workers; startup reclaim backstops SIGKILL (OPS-02, RESEARCH Shutdown) |
| T-04-12 | DoS (self) | a stuck task hangs graceful shutdown forever | mitigate | bound total shutdown join with `tokio::time::timeout` (Pitfall 8) |
| T-04-13 | Information Disclosure | DB URL/password leaked via startup/config-echo logs | mitigate | never log `database_url`; redacted config echo (Security Domain T-03-04) |
| T-04-14 | Information Disclosure | /metrics exposed publicly | mitigate | default `metrics_addr = 127.0.0.1`; binding wider is an explicit operator choice documented in config.example.toml (Security Domain V4) |
</threat_model>
