<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| signal/cancel → loop | Cancellation crosses into the running crawl; mishandling orphans leases |
| Postgres ← sampler aggregates | Repeated aggregate reads can starve the crawl if too frequent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Tampering/Repudiation | shutdown orphans in_progress leases | mitigate | Cancel at claim boundary + drain in-flight workers via join_worker; zero in_progress asserted by `graceful_drain_no_orphan_leases` (Pitfall 3) |
| T-04-09 | Tampering | select! aborts an in-flight apply_follow_list txn | mitigate | Never wrap process_batch in select!; cancel only at claim boundary (Pitfall 4) |
| T-04-10 | DoS (self) | sampler aggregate runs too often, starves crawl | mitigate | Coarse configurable sample interval (15-60s); single GROUP BY (Pitfall 6) |
</threat_model>
