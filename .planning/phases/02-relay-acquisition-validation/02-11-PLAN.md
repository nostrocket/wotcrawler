<objective>
Close BLOCKER 2 (WR-03 residual, RELAY-04): `fetch_complete_with_timeout`
(~line 273) does `let relay_url = pool_label(client).await;` — a joined string of
ALL connected relay URLs — and uses it as BOTH the `registry.acquire()` key and
the `FetchTimeout` label. This collapses per-relay GCRA quotas into one shared
pool quota (defeating RELAY-04) and mints a fresh full-burst limiter whenever pool
membership changes. `acquire_validated_lists_client` (src/relay/mod.rs:205)
already holds the correct individual `relay_url` but does NOT thread it into
`fetch_complete` (line 227).

Purpose: make per-relay rate limiting actually per-relay in production. Thread the
caller's real `relay_url` through the fetch path so each relay drives its own
independent GCRA limiter (one key per relay), preserving accrued GCRA state across
pool-membership changes and honoring RELAY-04's politeness contract.

Output: `fetch_complete` / `fetch_complete_with_timeout` taking an explicit
`relay_url` parameter (threaded from `acquire_validated_lists_client`), `pool_label`
demoted to diagnostics only, a small test-observable introspection method on
`RateLimiterRegistry`, and a new test proving two pooled relays get two independent
limiter keys.
</objective>
